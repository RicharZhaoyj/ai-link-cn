# AI.link.cn 内容发布标准指南

## 🎯 核心理念

### 我们的目标：
1. **为用户提供价值** - 帮助用户做出明智的工具选择
2. **保持客观中立** - 提供平衡的优缺点分析
3. **建立专业权威** - 成为AI工具领域的可信信息来源
4. **内容质量优先** - 专注于深度分析和实用建议

### 核心价值观：
- ✅ **真实性** - 基于实际使用体验
- ✅ **实用性** - 提供具体的使用建议
- ✅ **全面性** - 覆盖功能、价格、适用场景等各个方面
- ✅ **可读性** - 内容结构清晰，易于阅读

## 📋 内容结构标准

### 1. 基础页面结构（所有评测必须包含）
```
1. 标题和摘要
2. 目录导航
3. 核心功能解析
4. 实际使用体验
5. 定价和价值分析
6. 适用人群分析
7. 优缺点总结
8. 竞品对比（可选）
9. 常见问题解答
10. 最终建议
```

### 2. 页面长度标准
| 页面类型 | 字数要求 | 图片要求 | 视频建议 |
|---------|----------|----------|----------|
| **基础评测** | 1500-2000字 | 3-5张 | 可选 |
| **深度评测** | 2500-3500字 | 5-8张 | 建议添加 |
| **对比评测** | 2000-3000字 | 4-6张 | 可选 |
| **快速指南** | 800-1200字 | 2-3张 | 建议添加 |

## 🔍 SEO优化标准

### 1. Meta标签规范
```html
<!-- 必须包含 -->
<title>[工具名称] 完全评测 - [主要卖点] | AI.link.cn</title>
<meta name="description" content="[60-150字的简要描述，包含主要关键词]">
<meta name="keywords" content="[工具名称], AI工具, [工具类型], 评测, 使用体验, 优缺点">

<!-- 建议包含 -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "[工具名称]",
  "description": "[简要描述]",
  "review": {
    "@type": "Review",
    "reviewRating": {
      "@type": "Rating",
      "ratingValue": "[评分]",
      "bestRating": "5"
    }
  }
}
</script>
```

### 2. 关键词策略
| 关键词类型 | 每篇文章数量 | 位置要求 |
|-----------|--------------|----------|
| **主关键词** | 1个 | 标题、URL、第一段、H2 |
| **二级关键词** | 3-5个 | H2、H3、正文 |
| **长尾关键词** | 5-10个 | 正文自然出现 |

### 3. 内容优化要点
- **标题标签**：H1只使用一次，H2用于主要章节，H3用于子章节
- **内部链接**：每篇文章至少3-5个内部链接
- **外部链接**：链接到官方网站、文档等权威来源
- **图片优化**：添加alt标签，压缩图片大小
- **移动端优化**：确保页面在移动设备上正常显示

## 🎨 内容质量标准

### 1. 内容原创性要求
```
✅ 必须原创 - 基于实际使用体验
✅ 可以引用 - 引用官方文档和用户反馈
✅ 需要验证 - 所有数据和功能描述必须准确
❌ 禁止抄袭 - 不得抄袭其他评测内容
❌ 禁止AI生成 - 主要内容必须人工撰写
```

### 2. 评测深度要求
每个评测必须包含以下维度：
- **功能解析**：详细介绍核心功能和使用方法
- **使用体验**：实际使用过程中的感受和发现
- **场景应用**：不同使用场景下的表现
- **成本分析**：价格、性价比、替代成本
- **适用建议**：明确告诉用户是否适合他们

### 3. 写作风格要求
```
语气：专业但不生硬，友好但不随意
格式：段落清晰，使用列表和表格提高可读性
术语：解释专业术语，使初学者也能理解
案例：使用实际案例说明问题
建议：提供具体的行动建议
```

## 📊 数据验证标准

### 1. 功能验证
```
- 必须亲自测试或找到可靠的使用证据
- 准确描述功能限制和边界条件
- 注明测试环境和版本信息
- 区分个人体验和普遍事实
```

### 2. 价格信息验证
```
- 直接从官方网站获取最新价格
- 注明价格获取日期
- 说明价格包含的服务内容
- 比较不同价格方案的性价比
```

### 3. 对比分析要求
```
- 客观公正，不刻意贬低竞品
- 使用具体数据和事实支撑对比
- 说明对比的标准和维度
- 帮助用户根据自身需求选择
```

## 🔄 更新维护标准

### 1. 内容更新频率
| 工具类型 | 内容更新频率 | 价格更新频率 |
|---------|--------------|--------------|
| 主流AI工具 | 每3-6个月 | 每月检查 |
| 新兴AI工具 | 每1-3个月 | 每月检查 |
| 基础工具 | 每6-12个月 | 每季度检查 |

### 2. 更新内容要求
```
- 检查功能是否有重大更新
- 验证价格信息是否仍然准确
- 更新用户反馈和评价
- 添加新的使用技巧和案例
- 在页面顶部添加更新日期
```

### 3. 过时内容处理
```
- 明显的错误信息立即修正
- 过时的价格信息更新为最新
- 已停用的功能标注"已停用"
- 不再推荐的工具移动到归档
```

## 🎯 审核标准流程

### 1. 内容创作流程
```
1. 选题确定 → 工具选择和市场调研
2. 内容策划 → 确定评测角度和结构
3. 信息收集 → 收集工具信息和用户反馈
4. 内容撰写 → 根据标准撰写内容
5. 内部审核 → 检查准确性和质量
6. 发布上线 → 优化后发布
7. 后续跟踪 → 监控用户反馈和数据
```

### 2. 审核检查清单
```
✅ 标题和描述是否吸引人
✅ 内容结构是否清晰
✅ 信息是否准确和最新
✅ SEO优化是否到位
✅ 图片和媒体是否适当
✅ 链接是否正确有效
✅ 移动端体验是否良好
✅ 是否有明显的错误或遗漏
```

### 3. 质量评分标准
| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 内容深度 | 30% | 是否深入分析工具功能和价值 |
| 准确性 | 25% | 信息是否准确和可靠 |
| 实用性 | 20% | 是否为用户提供实际帮助 |
| 可读性 | 15% | 是否易于阅读和理解 |
| SEO优化 | 10% | 是否优化搜索引擎表现 |
